Assignment 1: dynamic dice rolling program

×

The first assignment that I choose to include in this portfolio is a dynamically-programmed dice rolling algorithm that calculates the number of ways a number, M, can be rolled with a given number of dice, N. The reasons I chose this project are several-fold: the project was done with another programmer, I had only learned the concept of dynamic programming days before, and it demonstrates my desire for efficiency, concision, and simplicity for code.

In real-world situations, it's rare to program an entire software system on one's own, especially software systems with substantial functionality. As such, my ability to work with a classmate and produce something of quality that we were both able to contribute to, to me, is something to be proud of. Furthermore, my ability to learn a new concept - in this case dynamic programming - understand it, and implement it in a limited timeframe demonstrates that I'm able to use my prior knowledge of programming to more easily acquire new methods for solving a problem and bring the solution to fruition, not only in a timely manner but while still maintaining high code standards. As such, the code is quite short but it's functionality is anything but. Initially, this code was at least 3 times as many lines, "spaghetti", and significantly less efficient. In general, my process for programming is to: understand the problem; prep the problem; work towards a solution and; once the solution is attained, improve the solution as much as I can so that it's concise, readable and simple.

The second assignment I chose to include in this portfolio is a minimax, alpha-beta pruning, and expectimax assignment I completed for my artificial intelligence class. I chose this project because it demonstrates my ability to work with relatively large codebases as well as use already written code and libraries to reach my goals, and it also demonstrates my ability to program with other people as this project was done in pairs as well.

The code here is as concise as I could have made it while maintaining readability. It also contains highly pythonic concepts that I learned to try to improve the code in terms of efficiency and concision. Additionally, I enjoyed this project specifically because minimax and it's derivatives are very common in the artificial intelligence world, therefore I am now able to read texts or code and understand the underlying processes as well as the general problem and solution to said problem. This has already proven to be useful in my recreational studies of algorithms; for example, StockFish, an artificial intelligence chess engine, uses minimax as well as depth-limited depth first search to calculate optimal moves and a numerical score for chess positions.
