Assignment 1: Dynamic Dice Rolling Program

×

The first assignment that I choose to include in this portfolio is a dynamically-programmed dice rolling algorithm that calculates the number of ways a number, M, can be rolled with a given number of dice, N. The reasons I chose this project are several-fold: the project was done with another programmer, I had only learned the concept of dynamic programming days before, and it demonstrates my desire for efficiency, concision, and simplicity for code.

In real-world situations, it's rare to program an entire software system on one's own, especially software systems with substantial functionality. As such, my ability to work with a classmate and produce something of quality that we were both able to contribute to, to me, is something to be proud of. Furthermore, my ability to learn a new concept - in this case dynamic programming - understand it, and implement it in a limited timeframe demonstrates that I'm able to use my prior knowledge of programming to more easily acquire new methods for solving a problem and bring the solution to fruition, not only in a timely manner but while still maintaining high code standards. As such, the code is quite short but it's functionality is anything but. Initially, this code was at least 3 times as many lines, "spaghetti", and significantly less efficient. In general, my process for programming is to: understand the problem; prep the problem; work towards a solution and; once the solution is attained, improve the solution as much as I can so that it's concise, readable and simple.

×

This project uses Q-Learning to "teach" PacMan to avoid ghosts and seek out food pellets. Q-Learning is fairly cutting edge technology in the world of AI, and this application is quite robust. I chose to include this because Q-Learning is still very applicable for AI applications, particularly in mixed environments.

Assignment 5: "Sleeping TA"

×

This project includes generic threading of "students" waiting on a "TA" to get help on a programming assignment in real time. The TA can only help 1 student at a time, and a max of 3 students can wait for help at a time, otherwise they return to programming for a random amount of time before attempting to get help again.

This served as the first time at Westminster that I programmed a somewhat sophisticated application that utilized multithreading, mutexes and semaphores. To that point, it's the first time that I've used semaphores at all. This served as a great opportunity to expand my knowledge of multithreading within the context of the Linux Operating system - this was a small scale simulation of virtual memory management in a similar way to how Linux does the same. This was coupled with investigations into Linux source code as inspiration for how it should be done in my application.

Senior Project: Chess Opening Trainer

×

To date, there isn't a simple and effective application to learn and practice openings on the internet. There are, of course, various ways to do so, but they're inconvenient, slow, expensive, highly restricted, or all of the above. For my senior project, I decided to put my own spin on an opening trainer which brings all of the best things about opening practice together in one place. The application is a simple webpage with various options for expanding one's repertoire. As far as I know, it's also the only site with real-time, responsive analysis from StockFish 11 running server-side for high-depth, quick analysis.

This project integrated nearly everything that I've learned in my computer science career in university. It touches on basically everything that a program may do from the ground up: it creates a child process running another application (which happens to be a high-level, minimax AI engine acquired from a personally modified API) and pipes I/O back and forth from that process. The I/O of this engine is sent across the network via a websocket, which I learned about by working with a friend, to the client and is displayed on the webpage. It uses several languages including HTML/CSS/JS, python, and several libraries within each of those languages.
